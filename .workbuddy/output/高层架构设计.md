## 1. 需求概要

### 1.1 需求概要说明

> 一句话描述本期需求要解决的核心业务问题、覆盖的业务场景与服务对象。

**业务现状**：AELA 是已上线的 Solo 模式桌面 Agentic Coding 助手（Electron 33 + React 18 + TypeScript，深度集成 `@agentprimordia/sdk` v1.0.0），具备 ReAct 自主编码、MCP 工具调用、多 Agent 编排、HITL 人工确认、记忆/RAG、安全护栏等能力，但安全基线（sandbox/IPC 校验/MCP ACL）与 SDK 依赖链路存在结构性缺口，且上游资料存在 12 处文档冲突（X1–X12）。

**触发事件**：本期由「冻结业务边界、对齐行业标杆（Cursor/Windsurf/Cline/Aider/Continue）、收敛 SDK 依赖与编排口径冲突」驱动启动，为下游系统/部署/安全/UserStory 提供统一边界基线。

**系统定位一句话**：AELA 定位为「本地优先、数据不出端的 Solo 桌面 Agentic Coding 引擎」，在桌面 GUI 内闭环完成自主编码—工具执行—人工确认—复盘留痕。
