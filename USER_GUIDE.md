## 1. 快速开始

### 1.1 安装

AELA 提供 Windows / macOS / Linux 安装包：

| 平台 | 安装包格式 |
|------|-----------|
| Windows | `.exe` (NSIS) |
| macOS | `.dmg` |
| Linux | `.AppImage` |

从 `release/` 目录获取对应平台的安装包，双击运行即可完成安装。应用启动后为单实例运行，不会同时打开多个窗口。

### 1.2 配置模型

首次使用前，需要配置至少一个 LLM 模型：

1. 点击侧边栏底部的 **⚙️ 设置**
2. 进入 **🤖 模型** 标签页
3. 点击 **添加模型**，填写以下信息：

| 字段 | 说明 |
|------|------|
| 名称 | 自定义模型名称，如 `GPT-4o` |
| Provider 类型 | `openai` / `anthropic` / `ollama` / `custom` |
| API Key | 对应平台的密钥 |
| Base URL | API 地址（Ollama 默认 `http://localhost:11434`） |
| 模型名称 | 如 `gpt-4o`、`claude-sonnet-4-20250514`、`qwen2.5-coder` |

4. 点击 **测试连接** 验证配置是否正确
5. 点击 **设为默认** 将其设为默认模型

> **支持的 Provider：**
> - **OpenAI** — 兼容 DeepSeek、Qwen、GLM 等所有 OpenAI 格式的 API
> - **Anthropic** — Claude 系列模型
> - **Ollama** — 本地部署的开源模型
> - **Custom** — 自定义 OpenAI 兼容端点

### 1.3 发起第一次对话

1. 点击侧边栏 **＋ 新建任务** 创建新会话
2. 在底部输入框中输入你的需求
3. 按 **Enter** 发送（Shift+Enter 换行），或点击发送按钮
4. AELA 将以流式方式实时输出回复，并展示工具调用过程

## 3. 对话与编码

### 3.1 两种模式

输入框支持两种工作模式切换：

| 模式 | 适用场景 | 推荐用途 |
|------|---------|---------|
| **Code（代码开发）** | 编程 / 调试 / 重构 / 架构 / Code Review | 应用开发、项目理解、游戏创意、工具脚本 |
| **Office（日常办公）** | 文档 / PPT / Excel / 邮件 / 会议纪要 | 网页分析、调研分析、数据挖掘、深度研究 |

不同模式会构建不同的系统提示词，影响 Agent 的行为方式。输入框上方提供 **智能推荐按钮**，点击可直接填充预设的 Prompt。

### 3.2 工作区

工作区是 Agent 操作文件的范围边界：

1. 在输入框点击 **选择工作目录**
2. 选择本地项目文件夹
3. Agent 将可以读取该目录下的文件、搜索代码、执行命令

**工作区功能：**
- 文件树浏览（自动过滤 `node_modules`、`.git` 等目录）
- 文件内容搜索（支持正则表达式）
- 文件读写（通过 `read_file` / `write_file` 工具）
- 项目结构获取

在设置 → 系统设置中可开启 **AGENTS.md** 和 **CLAUDE.md** 上下文文件，Agent 会自动读取项目根目录的这些文件作为项目上下文。

### 3.3 技能系统

Skills 是 Markdown 格式的技能文件，可以增强 Agent 的能力：

**使用方式：**
- 在输入框中输入 `/技能名称` 或 `/触发命令` 快速激活技能
- 点击输入框的 **技能** 按钮从列表中选择
- 激活的技能会在会话中持续生效

**Skill 文件格式：**
```markdown
---
name: 技能名称
description: 技能描述
tags: [tag1, tag2]
trigger: /my-skill
as_tool: true
tool_name: my_skill
tool_description: 工具描述
---

Skill 正文内容（Markdown）
```

**扫描路径：**
- `~/.aela/skills/` — 用户全局技能
- `<workspace>/.aela/skills/` — 工作区技能
- `~/.claude/skills/` — 第三方（Claude）
- `~/.codex/skills/` — 第三方（Codex）
- `~/.cursor/skills/` — 第三方（Cursor）

详见 [技能管理](#4-技能管理) 章节。

### 3.4 Shell 命令安全确认

当 Agent 需要执行 Shell 命令时，系统会根据风险评估自动处理：

| 风险等级 | 说明 | 示例 | 处理方式 |
|---------|------|------|---------|
| 🟢 安全 | 只读操作 | `git status`、`ls`、`cat`、`npm list` | 直接执行 |
| 🟡 中等 | 写操作 | `npm install`、`git commit`、`mkdir` | 弹窗确认 |
| 🔴 危险 | 高危操作 | `rm -rf`、`sudo`、`git push --force` | 必须明确确认 |

**确认选项：**
- **拒绝** — 不执行该命令
- **允许本次** — 仅本次执行
- **本次会话允许同类命令** — 会话级缓存，后续同类命令自动放行

### 3.5 人机协作 (HITL)

HITL（Human-In-The-Loop）允许你在 Agent 执行过程中进行人工干预：

- **中断点配置** — 在特定工具调用前暂停，等待人工确认
- **自动批准工具** — 配置信任的工具，自动放行
- **预算超限中断** — 当成本超出预算时自动暂停

当 Agent 触发中断时，会弹出审批弹窗，你可以选择 **通过** 或 **拒绝**。

配置路径：设置 → 人机协作

### 3.6 文件变更与 Diff 审查

当 Agent 修改文件时，对话界面顶部会出现 **📝 N 变更** 按钮：

1. 点击按钮展开 Diff 面板
2. 查看每个文件的具体修改内容
3. 对每个变更选择 **接受** 或 **拒绝**
4. 接受的变更将写入文件，拒绝的变更将被回滚

### 3.7 内嵌终端

对话界面顶部提供 **▣ 终端** 按钮，点击可在底部展开内嵌终端：

- 支持在当前工作区路径下执行命令
- xterm.js 终端模拟器，支持颜色、光标等
- 可在 Agent 运行时同时使用

### 3.8 提示词变体

AELA 内置 **6 种提示词变体**，让 Agent 展现不同专业角色：

| 变体 | 说明 | 适用场景 |
|------|------|---------|
| `default` | 平衡版本：完整工具指南 + 质量标准 + 自检 | 日常通用 |
| `concise` | 极简版本：节省 token | 小模型 / 长任务 |
| `safety-first` | 安全优先：强制验证 + 保守变更 | 生产环境 |
| `code-reviewer` | 代码审查：分级问题反馈（Blocker/Major/Minor） | 代码审查 |
| `pair-programmer` | 结对编程：边做边讲 + 小步推进 | 学习 / 协作开发 |
| `mentor-coach` | 导师教练：因材施教 + 建设性反馈 | 教学场景 |

配置路径：设置 → 系统设置 → 提示词风格变体

## 4. 技能管理

**入口：** 侧边栏 → 技能

技能管理页提供：

| 功能 | 说明 |
|------|------|
| 技能列表 | 展示所有已扫描到的技能，支持搜索过滤 |
| 内容预览 | 点击技能查看 Markdown 渲染内容（主题自适应） |
| 扫描路径 | 展示所有扫描目录及其来源 |
| 扫描诊断 | 每条路径的条目数、找到的技能数、加载错误 |
| 重新扫描 | 手动触发重新扫描技能目录 |

**创建自定义技能：**

1. 在 `~/.aela/skills/` 目录下创建子目录
2. 在子目录中创建 `SKILL.md` 文件
3. 按照上述格式编写 YAML frontmatter + Markdown 正文
4. 在技能管理页点击 **重新扫描**

**Skill 作为工具：**

当 Skill 的 `as_tool: true` 时，会被注册为 Agent 可调用的工具（工具名前缀 `skill_`），Agent 可以主动调用该技能获取指导。

## 5. 自动化任务

**入口：** 侧边栏 → 自动化

自动化任务允许你创建可重复执行的 Agent 任务：

**任务触发类型：**

| 类型 | 说明 |
|------|------|
| 手动触发 | 随时点击运行 |
| 定时触发 | Cron 表达式定时执行 |
| 事件触发 | 特定事件发生时自动执行 |

**操作：**
- **创建任务** — 定义任务名称、Prompt、触发方式
- **试运行** — 将 Prompt 传递到对话页面执行
- **查看历史** — 查看每次执行的记录和结果
- **启停切换** — 暂时禁用/启用任务

执行历史保留最近 200 条记录。

## 6. 多 Agent 编排

**入口：** 设置 → 功能模块 → 编排

AELA 提供强大的多 Agent 编排能力，支持 5 种编排模式。

### 6.1 基本编排（Pipeline / Parallel / Handoff / Pool）

| 模式 | 语义 | 适用场景 |
|------|------|---------|
| **Pipeline** | Agent 串行执行，前一个输出作为后一个输入 | 多步骤流水线处理 |
| **Parallel** | 所有 Agent 同时执行各自独立输入 | 多角度独立分析 |
| **Handoff** | 多个 Agent 轮流处理同一任务，传递上下文 | 复杂任务协作 |
| **Pool** | 池化调度，支持最大并发数控制 | 批量任务处理 |

**配置方式：**
1. 选择编排模式
2. 添加 Agent（指定名称、模型、系统提示词、输入、最大轮次）
3. 点击运行，实时查看流式事件
4. 每个 Agent 的开始、token 输出、完成状态都会实时展示

### 6.2 DAG 调度器

DAG（有向无环图）调度器支持：
- 定义步骤间的依赖关系
- 拓扑排序自动执行
- 无依赖的步骤并发执行
- Fail-Fast 策略（任一步骤失败则终止）

### 6.3 协作模式

多 Agent 协作支持 4 种模式：

| 模式 | 说明 |
|------|------|
| Debate | 多 Agent 辩论，通过对抗产出更优结果 |
| Review | Agent 互相审查对方输出 |
| Consensus | 共识达成模式 |
| Brainstorm | 头脑风暴模式 |

### 6.4 Supervisor 监督者

Supervisor 模式模拟团队管理者：
- **Worker 池** — 管理多个工作 Agent
- **任务分配策略** — 轮询 / 负载均衡 / 技能匹配
- **优先级队列** — 按优先级处理任务

### 6.5 动态拓扑 DAG

动态 DAG 在静态 DAG 基础上增加：
- **条件路由边** — 根据运行时条件选择执行路径
- **三种节点类型** — Agent 节点 / Transform 节点 / Condition 节点
- **动态拓扑** — 运行时决定执行流程

## 7. 开发者工具集

### 7.1 代码审查

**入口：** 设置 → 功能模块 → 代码审查

- 输入要审查的文件路径（每行一个）
- Agent 自动分析代码质量
- 按 **严重程度** 分级展示问题：🔴 Critical / 🟠 Error / 🟡 Warning / 🔵 Info
- 按 **类别** 分类：安全 / 性能 / 风格 / Bug
- 支持按文件分组查看
- 查看历史审查记录

### 7.2 自动测试生成

**入口：** 设置 → 功能模块 → 测试生成

- 输入源文件路径
- **分析** — 解析文件结构、函数、依赖关系
- **生成** — 自动生成测试代码
- **运行** — 直接执行生成的测试，查看通过/失败数

### 7.3 Repo Wiki 文档生成

**入口：** 设置 → 功能模块 → Repo Wiki

- 基于工作区项目自动生成结构化 Wiki 文档
- 包含项目架构、模块职责、关键类与函数说明、依赖关系等
- 左侧列表展示所有已生成的 Wiki
- 支持 Markdown 渲染预览

### 7.4 图片转代码

**入口：** 设置 → 功能模块 → 图片转代码

- 选择本地图片（设计稿 / 截图 / UI 原型）
- 选择目标框架：React / Vue 3 / HTML/CSS / Tailwind / Svelte / Angular
- 自定义组件名称和额外指令
- 支持响应式布局选项
- 生成后可基于反馈 **迭代优化** 代码
- 多文件输出展示

### 7.5 多文件编辑器

**入口：** 设置 → 功能模块 → 多文件编辑

- 同时编辑多个文件
- 文件树导航
- 代码编辑器

### 7.6 浏览器预览

**入口：** 设置 → 功能模块 → 浏览器预览

- 内嵌浏览器预览网页效果
- 支持指定 URL 或本地文件路径

### 7.7 Agent 配置

**入口：** 设置 → 功能模块 → Agent 配置

- 创建自定义 Agent 配置（名称、模型、系统提示词、可用工具）
- 可选工具：`read_file`、`write_file`、`list_directory`、`search_code`、`execute_command`、`web_fetch`、`mcp_tool`
- 在编排和 Sub-Agent 中复用这些配置

### 7.8 Sub-Agent 隔离

**入口：** 设置 → 功能模块 → Sub-Agent 隔离

- 定义多个 Sub-Agent 并行执行
- **资源配额** — 每个 Agent 可设置独立的资源限制
- **聚合模式** — concat（拼接）/ best（选优）/ merge（合并）/ vote（投票）
- **Fail-Fast** — 任一 Agent 失败时是否终止全部
- **最大并发数** — 控制并行度
- 从预设模板快速创建 Agent

## 8. 高级配置

### 8.1 成本与预算

**入口：** 设置 → 成本与预算

- **成本摘要** — 总成本、调用次数、总 Token、输入/输出 Token
- **按模型统计** — 每个模型的独立消耗
- **预算配置** — 设置总成本上限、单次 Token 上限、会话 Token 上限
- **模型定价表** — 自定义每个模型的单价
- **调用记录** — 查看最近的 LLM 调用明细
- 预算超限时自动触发 HITL 中断

### 8.2 上下文窗口管理

**入口：** 设置 → 上下文管理

| 策略 | 说明 |
|------|------|
| **Trim（裁剪）** | 移除最早的消息，保留最近 N 条 |
| **Compress（压缩）** | 使用 LLM 压缩历史消息为摘要 |

- 配置最大保留消息数
- 配置压缩触发阈值
- 手动执行裁剪/压缩

## 10. 常见问题

### Q: 首次启动后没有模型可用？

A: 点击侧边栏底部的 **设置**，进入 **模型** 标签页添加模型配置。至少需要一个可用模型才能开始对话。

### Q: Agent 无法访问我的项目文件？

A: 在对话输入框中点击 **选择工作目录**，选择你的项目文件夹。Agent 只能访问已选工作区内的文件。

### Q: Shell 命令被拦截怎么办？

A: AELA 会对 Shell 命令进行风险评估。中等和危险级别的命令需要你手动确认。你可以选择"允许本次"或"本次会话允许同类命令"。

### Q: 如何减少 Token 消耗？

A: 以下方式可以帮助降低成本：
- 选择 `concise` 提示词变体（设置 → 系统设置）
- 配置上下文窗口管理策略（设置 → 上下文管理）
- 设置预算上限自动中断（设置 → 成本与预算）
- 使用本地 Ollama 模型（免费）

### Q: 技能没有被扫描到？

A: 在技能管理页查看 **扫描诊断信息**，检查：
1. 技能目录是否包含 `SKILL.md` 文件
2. 目录路径是否在扫描范围内
3. YAML frontmatter 格式是否正确
4. 点击 **重新扫描** 重试

### Q: 多 Agent 编排和普通对话有什么区别？

A: 普通对话是单个 Agent 自主执行（Solo 模式）。多 Agent 编排允许多个 Agent 协作完成复杂任务，支持 Pipeline（串行）、Parallel（并行）、Handoff（轮替）、Pool（池化）等模式。

### Q: 数据存储在哪里？

A: 所有数据以 JSON 文件形式存储在 Electron 的 `userData` 目录下：
- `aela-config.json` — 应用配置、模型、工作区、MCP
- `aela-sessions.json` — 会话和消息
- `aela-automation.json` — 自动化任务
- `logs/audit.jsonl` — 审计日志

### Q: 如何完全重置应用？

A: 删除 `userData` 目录下的所有 `aela-*.json` 文件，重启应用即可恢复初始状态。
