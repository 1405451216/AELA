# Esc 中断机制设计文档

> 日期: 2026-07-07 | 状态: 已批准待实现

## 问题

停止生成只能通过点按钮，用户直觉按 Esc 无反应，体验断裂。

## 方案

Esc 在生成中触发"双按中断"——首次提示，2 秒内第二次才停。

## 行为

| 状态 | 行为 |
|------|------|
| 生成中 + Esc 第 1 次 | 显示小 toast "再次按 Esc 停止生成"，2 s 超时 |
| 生成中 + Esc 2 次内 | 立即 `handleStop()` 中断，隐藏 toast |
| 超时后 | escCount 归零，重新进入待触发态 |
| 面板打开 + Esc | 原有面板关闭链路优先，不进入中断 |

## 实现要点

- 仅 `isStreaming === true` 时监听
- 通过 keydown 事件屏蔽输入框内的默认行为（如取消选区）
- 与现有浮动面板协同时：先判断有无处理中的面板焦点，有则面板优先
- 计数器与定时器用 ref 持有，不触发重渲染

## 视觉

- Toast 位置：输入框正上方居中偏上
- 样式：半透明深色底 + 小字，匹配 dark card
- 动效：0.25s 淡入淡出
- 可选：底部倒计时细线

## 文件

- `ChatView.tsx` — 新增 escCount ref、定时器、监听
- `src/renderer/src/components/chat/EscInterruptToast.tsx` — 自包含纯 UI

## 风险

- 与现有 useKeyboardShortcuts 冲突风险：需在分派链路加"面板先关"短路判断
- 定时器内存泄漏风险：组件卸载时清除定时器

## 测试策略

1. 验证 2 次 Esc 内能中断
2. 验证超时后 Esc 不中断
3. 验证 Esc 不干扰面板关闭
4. 卸载组件后定时器已清除
